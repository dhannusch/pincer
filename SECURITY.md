# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Report vulnerabilities privately via DM to @dennishannusch on X (will get a proper email address for the project soon).

If email is unavailable, open a private security advisory in GitHub.

## What to Include

- Affected component(s)
- Reproduction steps or proof of concept
- Impact assessment
- Suggested mitigation (if known)

## Response Expectations

- Initial acknowledgement: within 3 business days
- Triage/update cadence: weekly until resolution

## Scope Notes

Sensitive areas include:

- Runtime authentication and signature verification
- Secret handling
- Adapter manifest validation and outbound request controls
- Pairing and credential exchange flow
